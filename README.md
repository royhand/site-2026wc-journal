# site-2026wc-journal

## 项目简介

这是一个用于归档与发布多个独立 HTML 页面的仓库。  
页面内容以静态 HTML 为主，不依赖特定域名或网站环境，适合作为轻量级的页面存档与展示空间。

## 目录说明

```
.
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放主要 HTML 页面
│   ├── page-01.html
│   └── ...
├── assets/             # 静态资源文件（图片、样式、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：各独立 HTML 页面存放目录，命名保持清晰有序。
- `assets/`：页面引用的资源文件，按类型归类存放。

## 页面归档说明

本仓库中的 HTML 页面为独立归档，彼此之间无强关联。  
每个页面均可单独访问，无需依赖后端或数据库。  
归档内容可能包含信息记录、资料整理、静态演示等类型，不限定具体用途。

## 维护说明

- 本仓库以静态页面为主，维护方式简单直接。
- 新增页面建议放入 `pages/` 目录，并在必要时更新入口页面。
- 资源文件请统一放置于 `assets/` 下，便于管理与引用。
- 不建议在页面中嵌入外部不可控资源，以保证归档的长期可访问性。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意 HTML 文件即可查看。
3. 也可通过 GitHub Pages 或其他静态托管服务发布。

## 许可

本项目仅用于页面归档与展示，不构成任何形式的服务或承诺。  
如需引用或分发，请保持页面内容完整，并注明来源。
